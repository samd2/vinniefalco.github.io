async_connect (4 of 4 overloads)

Asynchronously establishes a socket connection by trying each endpoint in a sequence, and terminating if a timeout occurs.

Synopsis

Defined in header <boost/beast/core/basic_stream_socket.hpp> 

template<
    class Protocol,
    class Executor,
    class Iterator,
    class ConnectCondition,
    class IteratorConnectHandler>
DEDUCED
async_connect(
    basic_stream_socket< Protocol, Executor >& s,
    Iterator begin,
    Iterator end,
    ConnectCondition connect_condition,
    IteratorConnectHandler&& handler);
Description

This function attempts to connect a socket to one of a sequence of endpoints. It does this by repeated calls to the underlying socket's async_connect member function, once for each endpoint in the sequence, until a connection is successfully established or a timeout occurs.

Parameters

Name

Description

s

The beast::basic_stream_socket to be connected. If the socket is already open, it will be closed.

begin

An iterator pointing to the start of a sequence of endpoints.

end

An iterator pointing to the end of a sequence of endpoints.

connect_condition

A function object that is called prior to each connection attempt. The signature of the function object must be: 

bool connect_condition(
    error_code const& ec,
    Iterator next);

handler

The handler to be called when the connect operation completes. Ownership of the handler may be transferred. The function signature of the handler must be: 

void handler(
    // Result of operation. if the sequence is empty, set to
    // net::error::not_found. Otherwise, contains the
    // error from the last connection attempt.
    error_code const& error,

    // On success, an iterator denoting the successfully
    // connected endpoint. Otherwise, the end iterator.
    Iterator iterator
);

Regardless of whether the asynchronous operation completes immediately or not, the handler will not be invoked from within this function. Invocation of the handler will be performed in a manner equivalent to using net::io_context::post().

Example

The following connect condition function object can be used to output information about the individual connection attempts: 

struct my_connect_condition
{
    bool operator()(
        error_code const& ec,
        net::ip::tcp::endpoint const& next)
    {
        if (ec) std::cout << "Error: " << ec.message() << std::endl;
        std::cout << "Trying: " << next << std::endl;
        return true;
    }
};

It would be used with the async_connect function as follows: 

std::vector<tcp::endpoint> endpoints = ...;
timed_stream s(ioc.get_executor());

async_connect(s, endpoints.begin(), endpoints.end(),
    my_connect_condition{}, connect_handler);
void connect_handler(
    error_code const& ec,
    std::vector<tcp::endpoint>::iterator)
{
    // ...
}
