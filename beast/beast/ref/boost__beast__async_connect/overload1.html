async_connect (1 of 8 overloads)

Asynchronously establishes a socket connection by trying each endpoint in a sequence, and terminating if a timeout occurs.

Synopsis

Defined in header <boost/beast/core/basic_timeout_stream.hpp> 

template<
    class Protocol,
    class Executor,
    class EndpointSequence,
    class RangeConnectHandler>
DEDUCED
async_connect(
    basic_timeout_stream< Protocol, Executor >& stream,
    EndpointSequence const& endpoints,
    RangeConnectHandler&& handler);
Description

This function attempts to connect a socket to one of a sequence of endpoints. It does this by repeated calls to the underlying socket's async_connect member function, once for each endpoint in the sequence, until a connection is successfully established or a timeout occurs.

Parameters

Name

Description

stream

The beast::basic_timeout_stream to be connected. If the underlying socket is already open, it will be closed.

endpoints

A sequence of endpoints. This this object must meet the requirements of EndpointSequence.

handler

The handler to be called when the connect operation completes. Ownership of the handler may be transferred. The function signature of the handler must be: 

void handler(
    // Result of operation. if the sequence is empty, set to
    // net::error::not_found. Otherwise, contains the
    // error from the last connection attempt.
    error_code const& error,

    // On success, the successfully connected endpoint.
    // Otherwise, a default-constructed endpoint.
    typename Protocol::endpoint const& endpoint
);

Regardless of whether the asynchronous operation completes immediately or not, the handler will not be invoked from within this function. Invocation of the handler will be performed in a manner equivalent to using net::io_context::post().
